Privacy Policy for DevSecAI Security Task Sync

Last Updated: August 2025
Effective Date: August 2025

1. Introduction

This Privacy Policy describes how DevSecAI ("we," "our," or "us") collects, uses, and protects your information when you use the DevSecAI Security Task Sync Forge app (the "App") for Atlassian Jira.

Important: We, not Atlassian, are responsible for the privacy, security, and integrity of your data when using our App.

2. Information We Collect

2.1 Jira Data

When you use our App, we access and process the following Jira data:

  • Project Information: Project names, keys, descriptions, and metadata
  • Issue Data: Issue titles, descriptions, status, priority, assignees, labels, and custom fields
  • User Information: User display names, email addresses (when available), and user IDs
  • Workflow Data: Issue transitions, status changes, and workflow configurations
  • Comments and Attachments: Issue comments and file attachments (when relevant to security tasks)

2.2 DevSecAI Platform Data

We synchronise the following data from your DevSecAI platform:

  • Security Tasks: Task names, descriptions, status, priority, and metadata
  • Application Information: Application names, descriptions, and security configurations
  • Team Data: Team names, member information, and organisational structure
  • Sync Records: Mapping between Jira issues and DevSecAI tasks

2.3 Technical Data

We collect technical information necessary for App functionality:

  • Installation Data: Forge installation ID, Jira site URL, and app configuration
  • Usage Analytics: App usage patterns, feature utilisation, and performance metrics
  • Error Logs: Technical logs for debugging and service improvement

3. How We Use Your Information

3.1 Primary Purposes

We use your information to:

  • Synchronise Tasks: Sync security tasks between Jira and DevSecAI platforms
  • Maintain Mappings: Track relationships between Jira issues and DevSecAI tasks
  • Provide Updates: Keep data consistent across both platforms
  • Enable Workflows: Support your security and compliance workflows

3.2 Secondary Purposes

We may also use your information to:

  • Improve the App: Analyse usage patterns to enhance functionality
  • Provide Support: Troubleshoot issues and provide customer assistance
  • Ensure Security: Monitor for security threats and unauthorised access

4. Data Storage and Security

4.1 Storage Location

Your data is stored in:

  • AWS Infrastructure: Our backend services run on Amazon Web Services (AWS) in the EU-West-2 (London) region
  • Atlassian Infrastructure: Some data remains within Atlassian's secure infrastructure
  • DevSecAI Servers: Synchronised data is stored on our secure DevSecAI platform

4.2 Security Measures

We implement industry-standard security measures including:

  • Encryption: All data is encrypted in transit using TLS 1.2+ and at rest using AES-256
  • Access Controls: Role-based access controls and authorisation mechanisms
  • Monitoring: Continuous security monitoring and threat detection
  • Compliance: Industry-standard security frameworks and best practices

4.3 Data Retention

  • Active Sync Data: Retained while the App is installed and actively syncing
  • Historical Records: Sync history retained for 24 months for audit purposes
  • Deleted Data: Permanently deleted within 30 days of App uninstallation

5. Data Sharing and Third Parties

5.1 We Share Data With:

  • DevSecAI Platform: Your synchronized data is shared with your DevSecAI account
  • AWS Services: Technical data is processed by our AWS infrastructure providers
  • Support Partners: Limited data may be shared with authorised support personnel

5.2 We Do Not Share Data With:

  • Marketing Companies: We do not sell or share your data for marketing purposes
  • Unauthorised Third Parties: We do not provide access to unauthorised entities
  • Competitors: We do not share your data with competing platforms
